架設網站的基礎知識

從靜態到動態：掌握網站部署的核心技術

在完成前四章的學習後，您已經具備了建立完整網頁應用程式的技能。 本章將帶您了解如何將這些技能轉化為實際可運行的網站， 學習靜態與動態網站的差異，以及各種部署技術和最佳實踐。

🎯 本章學習目標

  • 理解靜態網站與動態網站的核心差異
  • 掌握網站部署的基本流程和技術選擇
  • 學習網域、主機和 DNS 的基本概念
  • 了解現代網站架設的最佳實踐
  • 實際部署一個完整的網站專案

靜態網站 vs 動態網站

了解不同類型網站的特性，是選擇合適技術棧的第一步。 讓我們從基礎概念開始，逐步深入各種網站架設技術。

📄

靜態網站 (Static Website)

定義

由純 HTML、CSS、JavaScript 組成的網站，內容在部署前就已確定，伺服器直接傳送檔案給瀏覽器。

特性

  • ✅ 載入速度快
  • ✅ 安全性高
  • ✅ 成本低
  • ✅ 易於維護
  • ✅ 可快取
  • ❌ 無伺服器端處理
  • ❌ 無即時資料更新

適用場景

  • 個人部落格
  • 公司官網
  • 作品集網站
  • 文件網站
  • 行銷頁面

技術棧

HTML/CSS/JS Jekyll Hugo Gatsby Next.js
⚙️

動態網站 (Dynamic Website)

定義

內容根據使用者請求、資料庫狀態或使用者行為動態生成的網站，需要伺服器端處理。

特性

  • 🔄 內容動態更新
  • 👤 使用者互動
  • 💾 資料庫整合
  • 🔐 使用者認證
  • 📊 即時資料

適用場景

  • 電商網站
  • 社交媒體
  • 內容管理系統
  • 線上應用程式
  • API 服務

技術棧

React/Vue/Angular Node.js Python/Django PHP/Laravel MySQL/PostgreSQL

🤔 如何選擇網站類型？

選擇靜態網站的情況：

  • 內容更新頻率低
  • 不需要使用者登入功能
  • 預算有限
  • 重視載入速度
  • 團隊技術能力有限

選擇動態網站的情況：

  • 需要使用者互動
  • 內容經常更新
  • 需要資料庫功能
  • 有複雜的業務邏輯
  • 需要即時功能

⚡ 靜態網站的優勢與限制

✅ 優勢
  • 極速載入：無需伺服器處理，直接傳送檔案
  • 高可用性：CDN 分散式部署，幾乎不會當機
  • 成本低廉：只需檔案儲存空間，無需運算資源
  • 安全性高：無伺服器端漏洞，無資料庫風險
  • SEO 友善：搜尋引擎可直接索引內容
❌ 限制
  • 無即時資料：內容更新需要重新部署
  • 無使用者互動：無法處理登入、表單提交等
  • 無個人化：所有使用者看到相同內容
  • 無動態功能：無法根據使用者行為改變內容
  • 維護複雜：大量內容更新時需要自動化工具

💡 何時選擇靜態網站？

✅ 適合靜態網站的情況：
  • 內容穩定：網站內容不常變動
  • 展示為主：主要是展示資訊，不需要複雜互動
  • 效能優先：需要極快的載入速度
  • 成本考量：預算有限，希望降低營運成本
  • SEO 重要：需要良好的搜尋引擎優化
  • 安全性要求：需要最高的安全性
❌ 不適合靜態網站的情況：
  • 即時資料：需要顯示即時更新的資料
  • 使用者系統：需要登入、註冊、個人化功能
  • 複雜互動：需要複雜的表單處理或資料操作
  • 內容管理：非技術人員需要經常更新內容
  • API 服務：需要提供 API 給其他應用程式使用
  • 即時通訊：需要聊天、通知等即時功能

🚀 現代靜態網站的進化

雖然傳統靜態網站無法處理伺服器端邏輯，但現代技術讓靜態網站變得更加強大：

🔧 靜態網站生成器 (SSG)

如 Jekyll、Hugo、Gatsby 等，可以在建置時從資料庫或 API 獲取資料，生成靜態檔案。

Gatsby 範例：
// gatsby-node.js
exports.createPages = async ({ graphql, actions }) => {
  const { data } = await graphql(`
    query {
      allMarkdownRemark {
        nodes {
          frontmatter {
            slug
          }
        }
      }
    }
  `);
  
  data.allMarkdownRemark.nodes.forEach(node => {
    actions.createPage({
      path: `/blog/${node.frontmatter.slug}`,
      component: require.resolve('./src/templates/blog-post.js'),
      context: { slug: node.frontmatter.slug }
    });
  });
};
⚡ 無伺服器函數 (Serverless Functions)

結合靜態網站和無伺服器函數，可以處理表單提交、API 呼叫等動態功能。

Netlify Functions 範例：
// netlify/functions/contact.js
exports.handler = async (event, context) => {
  if (event.httpMethod !== 'POST') {
    return { statusCode: 405, body: 'Method Not Allowed' };
  }
  
  const { name, email, message } = JSON.parse(event.body);
  
  // 處理表單資料
  await sendEmail({ name, email, message });
  
  return {
    statusCode: 200,
    body: JSON.stringify({ message: 'Email sent successfully' })
  };
};

🔄 客戶端渲染 (CSR)

使用 JavaScript 在客戶端動態載入內容，結合靜態網站的優勢。

JavaScript 動態載入範例：
// 在靜態網站中使用 JavaScript 載入動態內容
async function loadUserData() {
  try {
    const response = await fetch('/api/user-data');
    const userData = await response.json();
    
    // 動態更新頁面內容
    document.getElementById('user-name').textContent = userData.name;
    document.getElementById('user-avatar').src = userData.avatar;
  } catch (error) {
    console.error('載入使用者資料失敗:', error);
  }
}

// 頁面載入時執行
document.addEventListener('DOMContentLoaded', loadUserData);

📋 為什麼靜態網站無法直接處理表單？
  • 無伺服器端處理：靜態網站只有 HTML、CSS、JavaScript 檔案，沒有伺服器端程式碼
  • 無法執行後端邏輯：無法處理資料庫操作、郵件發送、資料驗證等
  • 安全性限制：無法在客戶端安全地處理敏感資料

Git 基礎知識與 GitHub Pages 部署

Git 是現代軟體開發不可或缺的版本控制工具，GitHub Pages 則是最受歡迎的靜態網站託管服務。 讓我們從基礎開始，一步步學會使用 Git 和 GitHub Pages 來部署靜態網站。

🚀 快速前往 GitHub Pages 部署教學 ↓

🔧 Git 基礎概念

📚 什麼是 Git？

Git 是一個分散式版本控制系統，用於追蹤檔案變更、協作開發和管理程式碼歷史。

🎯 Git 的主要優勢：
  • 版本控制：記錄每次變更，可以隨時回到之前的版本
  • 協作開發：多人可以同時開發，自動合併變更
  • 分支管理：可以建立不同的開發分支，獨立進行功能開發
  • 備份保護：程式碼分散儲存在多個地方，避免資料遺失
  • 歷史追蹤：詳細記錄誰在什麼時候做了什麼變更

🏗️ Git 基本架構

📁 工作目錄 (Working Directory)

你正在編輯的檔案，也就是你的專案資料夾

↓
📋 暫存區 (Staging Area)

準備提交的檔案，使用 git add 將檔案加入

↓
💾 本地儲存庫 (Local Repository)

使用 git commit 將變更永久儲存到本地

↓
☁️ 遠端儲存庫 (Remote Repository)

使用 git push 將變更推送到 GitHub 等平台

🔧 網路基礎知識

🌐 HTTP/HTTPS 協定

HTTP (HyperText Transfer Protocol)

  • 應用層協定，用於網頁傳輸
  • 無狀態協定，每次請求獨立
  • 使用 TCP 作為傳輸層協定
  • 預設使用 80 埠號

HTTPS (HTTP Secure)

  • HTTP + SSL/TLS 加密
  • 使用 443 埠號
  • 提供資料加密和身份驗證
  • 防止中間人攻擊
HTTP 請求範例：
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)
Accept: text/html,application/xhtml+xml
Accept-Language: zh-TW,zh;q=0.9,en;q=0.8
Connection: keep-alive

🔗 TCP/IP 協定

TCP (Transmission Control Protocol)

  • 可靠的傳輸協定
  • 確保資料完整性和順序
  • 使用三次握手建立連線
  • 流量控制和擁塞控制

IP (Internet Protocol)

  • 網路層協定
  • 負責資料包路由
  • IPv4: 32 位元位址 (如 192.168.1.1)
  • IPv6: 128 位元位址 (如 2001:db8::1)
TCP 三次握手：
1. Client → Server: SYN (同步請求)
2. Server → Client: SYN-ACK (同步確認)
3. Client → Server: ACK (確認)

連線建立完成，開始資料傳輸

🔗 網址與 IP 位址的關係

📋 基本概念

網址（URL）和 IP 位址是網際網路中兩個不同層面的識別方式：

  • 網址（URL）：人類可讀的網域名稱，便於記憶和使用
  • IP 位址：電腦可理解的數值位址，用於實際的網路通訊
🏠 生活類比
📧 郵件地址

網址就像：「台北市信義區信義路五段7號」

IP 位址就像：「25.123456, 121.567890」（經緯度座標）

人們習慣用地址，但郵差需要知道確切座標才能送達

📞 電話號碼

網址就像：「王小明」

IP 位址就像：「0912-345-678」

我們記住姓名，但電話系統需要號碼才能接通

網址容易記憶和輸入，比 IP 位址更直觀

🔄 靈活性

可以輕鬆更換伺服器 IP，而不需要更改網址

🌍 全球性

網址可以在全球任何地方使用，不受 IP 位址限制

🏢 品牌識別

網址可以反映公司或組織的品牌形象

實作範例：部署靜態網站

📋 部署流程概覽

我們將使用 GitHub Pages 來部署我們的中西醫筆記網站，這是最簡單且免費的靜態網站部署方式。

1

準備專案檔案

確保您的專案包含以下檔案：

📁 專案根目錄
📄 index.html
📄 lesson_0.html
📄 lesson_1.html
📄 lesson_2.html
📄 lesson_3.html
📄 lesson_4.html
📄 lesson_5.html
📁 css/
📄 index.css
📄 lesson_0.css
📄 lesson_1.css
📄 lesson_2.css
📄 lesson_3.css
📄 lesson_4.css
📄 lesson_5.css
📁 img/
📁 js/
📁 pages/
2

建立 GitHub 儲存庫

Git 指令：
# 初始化 Git 儲存庫
git init

# 添加所有檔案
git add .

# 提交變更
git commit -m "Initial commit: 中西醫筆記網站"

# 連接到 GitHub 儲存庫
git remote add origin https://github.com/您的帳號/專案名稱.git

# 推送到 GitHub
git push -u origin main
3

啟用 GitHub Pages

  1. 進入您的 GitHub 儲存庫頁面
  2. 點擊「Settings」標籤
  3. 在左側選單中找到「Pages」
  4. 在「Source」部分選擇「Deploy from a branch」
  5. 選擇「main」分支和「/ (root)」資料夾
  6. 點擊「Save」
  7. 等待幾分鐘讓 GitHub 部署您的網站
注意：您的網站將在 https://您的帳號.github.io/專案名稱 上線
4

自訂網域 (可選)

如果您有自己的網域，可以設定自訂網域：

  1. 在 GitHub Pages 設定中輸入您的網域
  2. 在您的網域註冊商處設定 CNAME 記錄
  3. 等待 DNS 傳播完成
CNAME 記錄設定：
# 在您的 DNS 設定中添加
www.yourdomain.com → 您的帳號.github.io

🎉 部署完成！

恭喜！您的中西醫筆記網站現在已經成功部署到網際網路上。您可以：

  • 分享您的網站連結給朋友
  • 在任何裝置上瀏覽您的網站
  • 繼續開發新功能並自動部署
  • 使用自訂網域讓網站更專業
